## Excluded by design (still open questions)

### Discussion moderation via MCP

**The decision.** `/api/admin/discussions*` uses `requireDiscussionModerator` → `requireDiscussionUser`, and the latter explicitly throws 401 on any `Authorization` header:

```ts
const authz = req.headers.get("authorization") ?? req.headers.get("Authorization")
if (authz && authz.length > 0) {
    throw new AuthFailure(401, {
        error: "Authorization headers are not accepted on discussion routes. Sign in instead.",
    })
}
```

This is deliberate. Discussion moderation has CSRF protection and a per-action audit log (`DiscussionModerationLog`) that records `actorId`. An admin API key (often handed to AI/automation) moderating without a human session would weaken both.

**Options if we revisit:**

- Add a Bearer-auth path to `requireDiscussionModerator` that requires the API key to be owned by a real `MODERATOR`/`ADMIN` user and writes `actorId` from the key's `createdById`. Smallest change; the audit log already accepts any userId.
- Carve a separate bearer-only route surface (e.g. `/api/admin/bearer/discussions/*`) that mirrors the moderation actions. Cleaner separation but doubles the surface.
- Stay session-only. Discussion moderation through MCP requires a thin local script that authenticates as the moderator, hits the routes with the session cookie, and returns the result.

This isn't an emergency — the admin UI works. Pick when the workflow pain is worth the auth-design conversation.

### Pagination + filters on `list_problems` / `list_articles`

The admin REST endpoints don't currently accept `cursor`/`limit` query params. Adding tool params would silently ignore them; this needs joint API + tool change. Tracked.

### Remote MCP transport at `mcp.learndatanow.com`

Today's stdio transport means each user must clone + `npm run build` to use the MCP server. A hosted streamable-HTTP MCP endpoint would unblock ChatGPT custom connectors and zero-install Claude.ai usage. The skill's Phase 2 recommendation for new MCP servers in 2026 is remote HTTP. Design conversation parked.
